## v1.1.0

### 终于不用自己翻 Release 了 | Highlight

好消息：您养的花会自己挪新盆子了。

现在你可以直接在设置里检查、下载并安装更新，不用再手动打开 GitHub Release、找安装包、下载、覆盖安装——这一串流程终于可以少点仪式感了。

#### 应用内更新，爽

新增的更新功能支持从 GitHub Release 和 MirrorChyan 获取版本信息，并提供自动检查、手动检查、下载、安装、取消等完整流程。

你也可以在设置中调整自动检查频率，或者填写 Mirror酱 CDK 来使用对应的下载源。哦对了，安装更新前，花笺会尝试保存未保存事件，您不必为忘记 Ctrl+S 慌张。

> [!IMPORTANT]
> 应用内更新链路改动较多，如果遇到异常，欢迎提交 [Issue](https://github.com/Achilng/floral-notepaper/issues)。

#### 关于关于页

新增的“关于”面板现在会展示版本信息、贡献者列表，以及 GitHub 仓库链接等项目相关入口。

顺手还加了一些随机 Tip。它们不一定能改变世界，但至少能让关于页面不再只是一个放版本号的盒子。

#### Markdown 预览更像 GitHub 了

Markdown 预览新增了 GFM Alerts 支持，代码块也会显示语言标签。

编辑器侧也补上了右键菜单、图片粘贴与拖拽、重做等体验改进。少一点打断，多一点“本该如此”。

#### Windows 更新链路上了不少保险

这次对 Windows 更新流程做了不少兜底，包括安装回滚、注册表备份、进程校验、递归备份问题处理，以及隐藏更新子进程控制台窗口等。

简单说，就是给更新流程多加了几层安全垫。虽然不敢说从此万无一失，但至少摔下来的姿势会讲究一些。

---

### No More Manual Release Hunting | Highlight

Good news: your floral can repot it in a newer pot now.

You can now check, download, and install updates directly from the settings page, instead of opening GitHub Releases manually, finding the right installer, downloading it, and running it yourself. That little ritual is finally becoming optional.

#### In-app updates are here

The new update system supports fetching release information from both GitHub Releases and MirrorChyan, with automatic checks, manual checks, download, install, and cancel flows.

You can also configure the check frequency in settings, or provide a MirrorChyan CDK to use the corresponding download source. Before installing an update, Floral Notepaper will try to save unsaved events first, no more Ctrl+S anxiety now.

> ⚠️ Note: the update system touches quite a few paths,. If something goes wrong, it's always welcome to submit a [issue](https://github.com/Achilng/floral-notepaper/issues).

#### About About page

A new About panel has been added, showing version information, contributors, and links to GitHub, feedback, and licenses.

There are also some random tips. They may not change the world, but at least the About page is no longer just a version-number container.

#### Markdown preview is closer to GitHub

Markdown preview now supports GFM Alerts, and code blocks now show language labels.

The editor also gains a context menu, image paste and drag-and-drop support, redo support, and other editing improvements. The goal is simple: fewer interruptions, more “this should have worked like this already”.

#### Windows updates got extra safety padding

The Windows update flow received several enhancement, including installer rollback, registry backup, process checks, recursive backup handling, and hiding the console window of update subprocesses.

In short, the update process now has more guardrails. Not a promise that nothing can ever go wrong, but at least the landing should be less dramatic.
